ADVANCED: ADD REAL KAFKA/ELASTICSEARCH (Optional - Requires Docker)
═══════════════════════════════════════════════════════════════════════════════

IF YOU HAVE DOCKER INSTALLED:

1. Start Kafka:
   docker run -d --name kafka-zk -p 2181:2181 \\
     -e ZOOKEEPER_CLIENT_PORT=2181 \\
     confluentinc/cp-zookeeper:latest
   
   docker run -d --name kafka -p 9092:9092 \\
     -e KAFKA_BROKER_ID=1 \\
     -e KAFKA_ZOOKEEPER_CONNECT=kafka-zk:2181 \\
     -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \\
     -e KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1 \\
     confluentinc/cp-kafka:latest

2. Start Elasticsearch:
   docker run -d --name elasticsearch \\
     -e discovery.type=single-node \\
     -e xpack.security.enabled=false \\
     -p 9200:9200 \\
     docker.elastic.co/elasticsearch/elasticsearch:7.15.0

3. The connectors will automatically connect when you deploy Scenario 4

═══════════════════════════════════════════════════════════════════════════════
IF SOMETHING DOESN'T WORK
═══════════════════════════════════════════════════════════════════════════════

Problem: "Connection refused" for REST API
Solution: Make sure setup_infrastructure.py is still running in Terminal #1

Problem: "Port 514 already in use"
Solution: Another syslog is running. Use: python send_syslog.py localhost 5514
         And update environment: $env:SYSLOG_PORT='5514'

Problem: Kafka/Elasticsearch showing connection errors
Solution: This is normal! They're optional. Just ignore those errors.
          Or install Docker and start them (see above).

Problem: Windows Event Log showing permission denied
Solution: Run PowerShell as Administrator for full access
          Or ignore - it's optional
